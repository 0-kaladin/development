NotePad Documentation

NotePad is a simple application for entering and saving character-only notes. It demonstrates the following Android concepts:

  • ListView objects with Cursor-based adapters.
  • Data storage and access with a ContentProvider.
  • Intent filters that handle multiple actions.
  • Activity objects that handle more than one Intent.
  • Sending an Intent by selecting an item from a ListView.
  • Sending data in an Intent.
  • Dynamic construction of menus.
  • Context menus.
  • Providing other applications as alternative actions for a data item.
  • Creating Live Folders
  • Setting up an application that does not use "save" or "exit". The application saves the user's work whenever focus moves to another task. It remains available until the system destroys it.

Functions

By default, NotePad displays a list of existing notes, and provides a menu that allows you to insert a new note. If you click an existing note, you can edit it, or select the menu to delete it or edit its title. If you context-click an existing note, you can also delete it. If you navigate away from a new note once you've entered text, NotePad automatically saves what you've entered.

Limitations

NotePad has limited functionality. It does not provide search/replace. It does not support HTML or any other type of markup. However, it does have copy/paste if it is running on a version of the Android platform that supports that functionality.

Provider

Other applications can use NotePad's content provider to store notes. The "contract" for the NotePad provider is documented in the Appendix.

Alternative actions

Other applications can also register themselves to handle NotePad notes. To do this, they must provide the necessary specifications in their manifest file. The specifications are listed in the Appendix.
