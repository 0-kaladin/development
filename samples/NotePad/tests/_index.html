This sample contains a test package that demonstrates ContentProvider testing and Activity testing. It is a test of the Note Pad sample application.

Test Package

The test package project is located in the tests/ subdirectory of the Note Pad project. It provides an example of the structure of a test package project, and it contains a example of a AndroidManifest.xml file for a test package. The package uses InstrumentationTestRunner as its test runner.

The manifest for this test package contains an <instrumentation> element that links the test package with the application under test. Specifically, the element's android:name attribute specifies that Android should use InstrumentationTestRunner to control the classes in this package. The android:targetPackage attribute targets com.android.example.notepad as the Android package that contains the application under test.

ContentProvider Test

The ContentProvider test suite is contained in the single test case class NotePadProviderTest. This class extends ProviderTestCase2 , which extends the JUnit TestCase class and also provides methods and objects that facilitate ContentProvider testing.

Although the sample test suite for ContentProvider testing uses a single test case class, you may use as many test case classes as you wish, and each class can contain as many test methods as you wish.

ContentProvider test methods

NotePadProviderTest contains the following methods:

  • Test setup: The setUp() method re-initializes the test environment (the test fixutre) before each test is run. The call to super.setUp() creates a test context and test resolver, and creates a local instance of the provider under test. By creating the provider, the super constructor also tests the provider's onCreate() method, which is called during the provider's initialization.

    The setUp() method then gets a handle to the local provider's underlying database. This is used to set up test data in succeeding tests.

  • Test tear down: The tearDown() method calls super.tearDown(), which cleans up left-over object references to prevent memory leaks.
  • Test database creation: The insertData() helper method adds test data to the local provider. Individual test methods call insertData() when they need to set up a test fixture that contains valid data. This data is not inserted during setUp(), because some tests expect an empty database.
  • Content provider unit tests: The test package shows how to unit test a content provider's getType, query, insert, delete, and update methods against all the content Uris that the provider offers. Only a subset of all the possible unit tests is included.

Activity Test

The Activity test suite is contained in the single test case class NotePadActivityTest. This class extends ActivityInstrumentationTestCase2, which provides the JUnit test framework and also provides instrumentation methods that facilitate Activity testing.

The NotePadActivityTest test class contains a single assertion, testActivityTestCaseSetUpProperly(). This assertion calls Activity.getActivity() and then fails if the result is null, indicating that the test was unable to start the Activity under test.
